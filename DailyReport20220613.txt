#JavaScriptの基礎
##Hello,world!
・JavaScriptをHTML文書内に記述する場合は<script>タグ内で記述する

・type属性は現在必須ではない。ただし、HTML5ではこの属性の意味が変わっている

・language属性は現在は使われておらず必要ない

・外部スクリプトはsrc属性でHTMLに記述する
　この際ファイルパスは絶対パスまたは相対パスどちらでも記述できる
　また、URLも可能

・原則として、最もシンプルなスクリプトだけをHTML内に置き、複雑なものは別ファイルへ置く

・scriptタグにsrcが設定された場合、scriptタグ内のコードは無視される

##コード構造
・JavaScriptは改行を暗黙のセミコロンと解釈しているため、他の言語とは異なりセミコロンがなくても動作する
　ただし、角括弧前にはセミコロンを想定していないなど、JavaScriptの想定外の記述をするとエラーが発生するので、基本的にはセミコロンを付けたほうがよい

・JavaScriptでは波括弧は新しい行ではなく、同じ行に書くエジプトスタイルを用いる
　また、開始のの括弧の前にはスペースがある

・バッククォートを使うと、文字列を複数行に分割できる

・コードを論理ブロックに分割するための空行を入れる
　例：変数の初期化、メインのループ、結果の返却

・コードをネストしすぎないようにcontinueを用いたりして工夫する

・関数を使用するコードが最初で、その後に関数を記述する

・Linterとはコードのスタイルを自動でチェックし改善が提案できるツールのこと
　未宣言変数へのアクセスなど、ユーザーのミスをハイライトしてくれる

##モダンなモード、"use strict"
・"use strict"もしくは'use strict'がスクリプトの先頭に位置する場合、すべてのスクリプトは最新の方法で動作する
　また、関数の頭にも置くことができ、その場合は関数内でのみStrictモードが有効になる
　一度Strictモードに入ったらキャンセルできない

・use strictでスクリプトを開始することを推奨している

##変数
・変数宣言は可読性を高めるため、1変数1行にする

・変数名は文字、数字、$、_のみしか使えない
　また、変数名の先頭は数字であってはいけない

・Strictモードでない場合にはletなしで、値を代入するだけで変数を作成することができるが、推奨されていない

・定数を宣言する場合にはconstを用いる
　また、定数の変数名には大文字とアンダースコアを用いる

・変数名はそれが何を表しているのかがわかりやすい名前を付ける

##データ型
・数値型は-(2^53-1) < x < (2^53-1)の範囲の数値を表現できる
　この範囲以上に大きな値を用いる場合にはBigInt型を用いる
　BigInt型は末尾にnを追加することで作成される

・ダブルクォートとシングルクォートには違いはない

・バッククォートは文字列の中に変数を埋め込む際に用いる

・JavaScriptではnullは無し、空、不明な値を表す

・undefinedは値が代入されていないことを表す

・typeof 引数で引数に渡した値の型を返値として返す

##インタラクション：alert,prompt,confirm
・alertを用いると、モーダルウィンドウを開き、引数で指定したメッセージを表示し、ユーザーがOKをクリックするのを待つ

・モーダルウィンドウのモーダルとはそのウィンドウを処理するまで訪問者はページの他の部分と対話したり、他のボタンを押すことができないことを意味する

・promptを用いると、モーダルウィンドウが開かれ、ユーザーからの入力を受け取れる
　テキストもしくは入力がキャンセルされた場合にはnullが返却される
　IEでpromptを用いる際にはdefault値が未設定の場合、undefinedを挿入するため、default値を引数で指定することが推奨される

・confirmを用いると、モーダルウィンドウが開かれ、ユーザーがOKを押した際にはtrueをそれ以外はfalseを返す

・alert,prompt,confirmには制限がある
　モーダルウィンドウの位置はブラウザによって決定される（通常は画面中央）
　ウィンドウの見た目もブラウザに依存し、変更できない

##型変換
・JavaScriptでは多くの場合、演算子と関数は自動的に値を正しい型に変換する
　ただし、明示的に変換する必要がある場合もある

・数値変換を行った際に文字列が有効な数値でない場合にはNaNを返す
　また、数値変換の際にundefinedはNaNになり、nullは0になる

・Boolean変換は直感的に空（0,空文字,null,undefined,NaNなど）の値はfalseになる
　ただし、文字列の"0"はtrueになる(JavaScriptでは、非空文字は常にtrue)

##演算子
・オペランドとは演算子が適用されるもののこと
　例：5*2では5と2がオペランド
　オペランドは引数とも呼ばれる

・負の単項演算"-"は数値の符号を反転する

・べき乗は**で表す

・二項演算子"+"が文字列に適用された場合は、文字の結合を行う
　例：2+2+"1"は"41"となる
　その他の算術演算子は数値でのみ動作し、常にオペランドを数値に変換する

・単項演算子"+"は数値の場合何も起こらないが、数値でない場合には数値に変換する
　
・演算子の優先順位は、単項演算子、べき乗、乗除、加減、代入の順で高い

・インクリメント、デクリメントの前置式は、値を増加させ、増加させた値を返す
　後置式は値を増加させ、増加前の値を返す
　例：counter=1の場合
    a = ++counter //a = 2
    a = counter++ //a = 1
    どちらもcounterの値は2

・ビット演算子は引数を32ビットの整数値として扱い、それらのバイナリ表現のレベルで処理する
　AND( & )
  OR( | )
  XOR( ^ )
  NOT( ~ )
  LEFT SHIFT( << )
  RIGHT SHIFT( >> )
  ZERO-FILL RIGHT SHIFT( >>> )

##比較
・比較演算子はブール値を返す
　そのため、result = 5 > 4というように変数にbool値を格納できる

・文字列比較は先頭から各文字を比較していき、どちらかが大きくなった時点で結果を返す
　同時に終わった場合は同じ文字として認識する
　そうでなければ長い文字列が大きい
　この際、Unicodeの値で比較している

・比較演算子の場合は、-や/と同様に文字列を数値として比較する

・===を用いると型変換なしで等価をチェックする
　例：(0 == false)はtrueだが、(0 === false)はfalseになる

・!==も存在する

・undefinedとnullの等価チェックはいずれの変換もされずに行われる

##条件分岐：if,'?'
・三項演算子の一番の左のオペランドでは丸括弧で囲む必要はないが、可読性をよくするために括弧を使うことが推奨される

##論理演算子
・JavaScriptではOR演算子は左から右にオペランドを評価し、結果がtrueならオペランドの本来の値を返す
　また、すべてのオペランドが偽のとき最後のオペランドの本来の値を返す

・AND演算子の場合は左から右にオペランドを評価し、結果がfalseならオペランドの本来の値を返す
　また、すぺてのオペランドが真のとき最後のオペランドのほんらいの値を返す
